## 4. 心理语义+杯型提示词库

### 4.1 概述

在用户输入中文设计需求（如"简约轻量高脚杯"）后，系统通过 `prompt_library.py` 模块进行**两阶段转译**，将中文描述映射为 Stable Diffusion 理解的英文描述：

```
用户中文输入 "简约轻量高脚杯"
    │
    ├── 阶段1：心理语义提取 ──→ 「简约」→ "simple composition..."
    │                           「轻量」→ "thin glass wall..."
    │
    ├── 阶段2：杯型匹配 ──→ 「高脚杯」→ "transparent goblet glass cup..."
    │
    ▼
    合并输出: "simple composition, thin glass wall, transparent goblet glass cup..."
```

### 4.2 心理语义映射表

| 中文语义词 | 英文画面描述 | 视觉影响 |
|-----------|-------------|----------|
| 简约 | `simple composition, centered object, plain solid background...` | 减少视觉密度 |
| 复杂 | `multi-layer structure, rich textures, abundant details...` | 增加细节层次 |
| 柔和 | `smooth curved lines, soft natural side light, soft shadow...` | 圆润光影过渡 |
| 硬朗 | `sharp straight edges, hard side lighting, harsh shadow...` | 锐利几何感 |
| 轻量 | `thin glass wall, slender cup body, slender base...` | 轻透感 |
| 重量 | `thick glass wall, heavy thick base, solid volume sense...` | 稳重厚实感 |
| 扩张 | `wide opening, outward stretched outline, open shape...` | 横向延伸 |
| 收敛 | `narrow rim, inward gathered outline, compact vertical shape...` | 纵向收紧 |

### 4.3 杯型匹配算法

**匹配策略**（4 级由精确到模糊）：

# 如输入「高脚」可匹配「高脚杯」「红酒高脚杯」「迷你高脚烈酒杯」等

# 步骤3：反向模糊匹配（杯型名是用户文本的子串）
# 如「简约高脚杯」匹配「高脚杯」

# 步骤4：兜底 — 返回原文

**支持 40 种杯型**，按 key 长度降序匹配，长词优先。

## 5. 图像特征提取（轮廓复杂度分析）

### 5.1 概述

从 SD WebUI 生成的玻璃杯图片中提取 7 项几何特征，核心输出为**轮廓复杂度 C**（0~1），用于 MOPSO 动态调整目标函数系数。

### 5.2 算法流程

```
base64 图片
    │ decode_b64()
    ▼
numpy 灰度数组 (H, W)
    │ 阈值分割（自动阈值）
    ▼
杯体二值 mask
    │ Canny 边缘检测
    ▼
前景边缘图
    │ 轮廓边界框
    ▼
特征计算
    │
    ├── 高宽比           aspect_ratio
    ├── 高度占比         height_ratio
    ├── 宽度占比         width_ratio
    ├── 边缘密度         edge_density
    ├── 对称度           symmetry
    ├── 轮廓变化率       contour_variation
    └── 轮廓复杂度 C     complexity（加权合成）
```

### 5.3 Canny 边缘检测（自实现，无需 OpenCV）

**步骤 1：高斯模糊**

```python
# 5×5 高斯核，σ=1.0，分离式卷积
K_gauss: 1D array [0.061, 0.244, 0.390, 0.244, 0.061]  # 归一化后
blurred = convolve_horizontal(gray, K_gauss)
blurred = convolve_vertical(blurred, K_gauss)
```

**步骤 2：Sobel 梯度**

```python
# Sobel 水平/垂直核
Kx = [[-1, 0, 1], [-2, 0, 2], [-1, 0, 1]]
Ky = [[-1, -2, -1], [0, 0, 0], [1, 2, 1]]

Gx = conv2d(blurred, Kx)
Gy = conv2d(blurred, Ky)
G = sqrt(Gx² + Gy²)           # 梯度幅值
θ = atan2(Gy, Gx + 1e-8)     # 梯度方向
```

**步骤 3：非极大值抑制（NMS）**

沿梯度方向比较邻近像素，只保留局部极大值，将边缘细化到单像素。

**步骤 4：双阈值连接**

```python
high_thresh = 60
low_thresh = 20

强边缘 = G >= high_thresh
弱边缘 = low_thresh <= G < high_thresh
# 弱边缘若与强边缘 8-邻域连通则保留，否则丢弃
```

### 5.4 杯体分割

```python
# 自适应阈值：取图片左侧 20 列灰度均值作为背景参考
bg_mean = mean(gray[:, :20])
threshold = max(80, bg_mean - 30) if bg_mean > 128 else 128
mask = gray < threshold
```

- 中值滤波（3×3）去噪
- 取最大连通区域作为主杯体（scipy `ndimage.label` / 回退版本）

### 5.5 特征计算公式

| 特征 | 符号 | 公式 | 含义 |
|------|------|------|------|
| 高宽比 | AR | `H_box / W_box` | >1 瘦高，<1 矮胖 |
| 高度占比 | HR | `H_box / H_img` | 杯体占图片高度比例 |
| 宽度占比 | WR | `W_box / W_img` | 杯体占图片宽度比例 |
| 边缘密度 | ED | `sum(foreground_edges) / sum(mask)` | 边缘细节丰富度 |
| 对称度 | S | `1 - mean(|left_mask - flip(right_mask)|)` | 0~1，越接近 1 越对称 |
| 轮廓变化率 | CV | `mean(std(diff(left_profile)), std(diff(right_profile))) / W_box × 10` | 杯体侧边起伏程度 |

**轮廓复杂度 C（加权合成）**：

```python
complexity = 0.50 × contour_variation
           + 0.20 × (1 - symmetry)
           + 0.30 × min(edge_density × 3, 1.0)
```

C 的范围被截断到 [0, 1]。

### 5.6 特征来源

- **C（轮廓复杂度）**：Sobel Canny 边缘检测 + 轮廓变化率分析
- **H（高度）**：`height_ratio` — 从轮廓边界框计算
- **D（宽度）**：`width_ratio` — 从轮廓边界框计算  
- **T（壁厚）**：当前未直接从图像提取（需边缘像素距离分析，待扩展）

### 5.7 边界处理

- 边缘点 < 10：返回安全默认值（C=0.5）
- 无 scipy：不执行连通区域去噪，直接使用 mask
- `_fallback_features()` 确保不会因单个图片失败导致整个请求崩溃

## 6. MOPSO 多目标粒子群优化算法

### 6.1 问题定义

**决策变量**（6 维）：玻璃热成型工艺参数

| 变量 | 符号 | 单位 | 默认区间 |
|------|------|------|----------|
| 加热温度 | `T_heat` | °C | [550, 700] |
| 加热时间 | `t_heat` | s | [30, 180] |
| 吹制压力 | `P_air` | MPa | [0.1, 0.5] |
| 吹制时间 | `t_blow` | s | [5, 30] |
| 冷却速率 | `R_cool` | °C/s | [10, 50] |
| 目标壁厚 | `D_wall` | mm | [2, 5] |

**优化目标**（4 项，全部最小化）：

| 目标 | 符号 | 物理含义 |
|------|------|----------|
| 壁厚均匀度 | f₁ | 变异系数 CV（越小越均匀） |
| 成品废品率 | f₂ | 比例 0~1（越小良品率越高） |
| 加工能耗 | f₃ | 加热+吹制+冷却能耗和 |
| 耐热安全性 | f₄ | 1 − 耐热指数（越小耐热越好） |

**约束**：每个变量在对应的 `[lb, ub]` 区间内（由特征复杂度 C 动态调整，见第 8 节）

### 6.2 算法参数

| 参数 | 值 | 物理意义 |
|------|-----|---------|
| `num_particles` | 50 | 种群规模 |
| `max_gen` | 100 | 最大迭代代数 |
| `w` | 0.5 | 惯性权重（保持原有搜索方向） |
| `c₁` | 1.5 | 个体学习因子（向自身最优靠近） |
| `c₂` | 1.5 | 社会学习因子（向全局最优靠近） |

### 6.3 数据结构

**Fitness 类**：多目标适应度容器

```
values: tuple[float]    # 原始目标值 (f₁, f₂, f₃, f₄)
weights: tuple[float]   # 权重（本系统全为 -1.0，最小化）
wvalues: tuple[float]   # 加权值 wᵢ × vᵢ
dominates(other) → bool # 帕累托支配判定
```

**Particle 类**（继承 list）：

```
Particle = [x₀, x₁, x₂, x₃, x₄, x₅]  # 6 维位置向量
 + speed: list[float]                   # 速度向量
 + fitness: Fitness                     # 当前适应度
 + best: Particle                       # 个体历史最优的克隆
```

**ParetoArchive 类**：非支配解归档，上限 100 条

```
update(population):        # 增量更新，自动去重
    for each individual:
        if archive 中存在支配 ind 的: 跳过
        if ind 支配 archive 中的: 移除被支配的
        不重复 → 克隆加入归档
_truncate():               # 超限时均匀抽稀
    按首目标排序 → 等距采样保留 max_size 条
```

### 6.4 优化主循环

```
算法：MOPSO（自实现）
输入：objective_functions, bounds, num_particles=50, max_gen=100, complexity=0.5
输出：pareto_front (ParetoArchive), None

1. 初始化
   for i = 1 to num_particles:
      x[i] = [random.uniform(lb_k, ub_k) for k = 1..6]
      v[i] = [random.uniform(-1, 1) for _ = 1..6]
      best[i] = None

2. 主循环
   for gen = 0 to max_gen - 1:

     2.1 评价
     for each particle:
         params = {param_name: x[i][k]} ∪ {complexity}
         fitness.values = (f₁(params), f₂(params), f₃(params), f₄(params))

     2.2 更新帕累托归档
     pareto_front.update(population)

     2.3 选择全局引导解
     global_best = random_choice(pareto_front.items)

     2.4 更新粒子
     for each particle:
         # 更新个体历史最优
         if best[i] is None or fitness(particle) > fitness(best[i]):
             best[i] = clone(particle)

         for k = 1 to 6:
             r₁, r₂ ~ U(0, 1)
             cognitive = c₁ × r₁ × (best[i][k] - x[i][k])
             social    = c₂ × r₂ × (global_best[k] - x[i][k])
             v[i][k]   = w × v[i][k] + cognitive + social

             # 速度限幅到变量范围的 ±10%
             max_speed = (ub_k - lb_k) × 0.1
             v[i][k]   = clamp(v[i][k], -max_speed, max_speed)

             # 更新位置
             x[i][k]  += v[i][k]

             # 边界截断（超出直接钳制）
             x[i][k]   = clamp(x[i][k], lb_k, ub_k)

     2.5 记录历史
     avg_fitness[gen] = mean(fitness.values over all particles)
     min_fitness[gen] = min(fitness.values over all particles)
```

### 6.5 帕累托支配判定

```
Particle A dominates Particle B ⟺
    ∀k: wᵏ(A) ≥ wᵏ(B)  AND  ∃k: wᵏ(A) > wᵏ(B)
```

其中 `wᵏ = weight_k × value_k`，本系统 `weight_k = -1.0`（最小化），等价于经典帕累托支配。

### 6.6 最优解选择（v2.4 归一化加权）

**v2.4 改进**：先归一化消除量纲差异，再加权综合评分（取最大）。

**背景问题**：四个目标值量级差异巨大：

| 目标 | 量级范围 | 直接加权的影响 |
|------|---------|---------------|
| 壁厚均匀度 f₁ | ~0.02～0.08 | 贡献占比几乎为零 |
| 成品废品率 f₂ | ~0.01～0.20 | 贡献很小 |
| 加工能耗 f₃ | **~5000～14000** | **主导加权结果** |
| 耐热安全性 f₄ | ~0～0.40 | 贡献极小 |

原始直接加权会导致**能耗几乎决定最终选择**，其他三个目标形同虚设。

**归一化方法（min-max）**：

```
对于 Pareto 前沿中的每个目标 k：
    min_k = 前沿中第 k 项的最小值
    max_k = 前沿中第 k 项的最大值

    归一化值 Sᵢ = (max_k - fᵢ) / (max_k - min_k)    # fᵢ 为第 i 个解的原始值
    当 max_k == min_k 时，Sᵢ = 1.0                     # 边界情况：全部相同时视为最优
```

归一化后所有目标映射到 [0, 1]，**1 表示该目标在当前位置最优**。

**等权加权综合评分**：

```python
weights = [0.25, 0.25, 0.25, 0.25]
score = Σ wᵢ × Sᵢ            # 归一化值加权求和
best_solution = argmax(score) # 取分最大者（综合最优）
```

> **⚠ 权重差异说明**：本节（6.6 最优解选择）使用 **等权 0.25**，目的是从 Pareto 前沿中挑选出四个目标综合表现最均衡的代表解，用于向用户展示"综合最优方案"。而第 [6.8 节](#68-量产可行性指数-piv24-新增) 的 PI 指数采用差异化权重（壁厚 0.30、废品率 0.35、能耗 0.15、耐热 0.20），其目的是评估该方案的**工业量产可行性**。两套权重用途不同，不应混淆。

### 6.7 历史记录

每代记录 4 个项目标的种群均值与最小值：

```python
iteration_history[gen] = {
    "avg_wall_uniformity":  mean(f₁) × 100,       # %
    "avg_defect_rate":      mean(f₂) × 100,       # %
    "avg_energy":           mean(f₃),              # 原始单位
    "avg_heat_resistance":  (1 - mean(f₄)) × 100,  # %
    "min_wall_uniformity":  min(f₁) × 100,
    ...
}
```

### 6.8 量产可行性指数 PI（v2.4 新增）

在 Pareto 最优解基础上，进一步计算**量产可行性指数 PI**，回答"该设计方案是否适合大规模工业量产"。

**计算公式**：

```
对四个目标分别进行 min-max 归一化（方法同 5.6 节）：
    S₁ = 壁厚均匀度归一化值（越大越好）
    S₂ = 废品率归一化值（越大越好）
    S₃ = 能耗归一化值（越大越好）
    S₄ = 耐热安全性归一化值（越大越好）

PI = w₁ × S₁ + w₂ × S₂ + w₃ × S₃ + w₄ × S₄
```

**权重配置**：

| 指标 | 权重 | 理由 |
|------|------|------|
| 壁厚均匀度 | 0.30 | 直接影响产品质量等级 |
| 成品废品率 | **0.35** | 权重最高，直接决定生产成本与企业收益 |
| 加工能耗 | 0.15 | 成本因素，但可通过电价改善 |
| 耐热安全性 | 0.20 | 安全性底线，但满足最低要求即可 |

**量产等级划分**：

| PI 范围 | 评价 | 说明 |
|---------|------|------|
| PI ≥ 0.85 | **非常易量产** | 各项目标均处最优区间 |
| 0.75 ≤ PI < 0.85 | **易量产** | 整体表现良好 |
| 0.60 ≤ PI < 0.75 | **可量产** | 部分指标需要关注 |
| 0.45 ≤ PI < 0.60 | **较难量产** | 多项指标偏离较远 |
| PI < 0.45 | **不建议量产** | 需重新调整设计 |

PI 值在 `/api/optimize` 返回结果中随 `pi` 和 `pi_grade` 字段返回，前端以绿色评分卡 + 等级标签展示。

## 7. 4 项目标函数（含轮廓复杂度 C 联动）

### 7.1 壁厚均匀度 f₁

**数学公式**：

```
temp_factor    = (T_heat - 550) / 150                           ∈ [0, 1]
pressure_factor = 1 - |P_air - 0.3| / 0.2                      ∈ [0, 1]
base_std       = 0.12 + C × 0.06                                # C 影响基线
uniformity_gain = (1 - C × 0.4) × (temp_factor × 0.08 + pressure_factor × 0.04)
std_dev        = max(0.02, base_std - uniformity_gain)
壁厚分布 ~ Normal(D_wall, std_dev, N=100)                       # rng(42) 固定种子
f₁ = std(壁厚) / mean(壁厚)                                     # 变异系数 CV
```

**物理意义**：
- 温度越高、压力越接近 0.3 MPa → 壁厚越均匀（标准差越小）
- C 越高 → 基础标准差增大（复杂形状更难均匀），且温度和压力均匀化收益降低

### 6.2 成品废品率 f₂（v2.4 平方惩罚）

**v2.4 改进**：绝对值偏离 → **平方偏离惩罚**。平方惩罚更真实地反映实际工艺规律——小偏差容忍度高，大偏差导致废品率快速上升。

**数学公式**：

```
base_rate        = 0.01 + C × 0.04                              # C=0 → 1%, C=1 → 4%
temp_sensitivity = 1.0 + C × 1.5                                # 温度敏感性 1.0~2.5x
time_sensitivity = 1.0 + C × 1.0                                # 时间敏感性 1.0~2.0x

temp_deviation   = |T_heat - 620|
time_deviation   = |t_heat - 90|

temp_penalty     = temp_deviation² × temp_sensitivity / 140000   # 平方惩罚
time_penalty     = time_deviation² × time_sensitivity / 270000   # 平方惩罚
pressure_penalty = (P_air - 0.3)² / 0.04 × 0.04                  # 平方惩罚

f₂ = min(base_rate + temp_penalty + time_penalty + pressure_penalty, 0.40)
```

**物理意义**：
- 以 620°C / 90s / 0.3 MPa 为工艺最佳点
- 偏离最佳点越远 → 废品率越高
- C 越高 → 废品基线升高 + 对偏离的敏感性增大（复杂杯型容错性差）

### 6.3 加工能耗 f₃

**数学公式**：

```
complexity_factor = 1.0 + C × 0.3                              # C=0 → ×1.0, C=1 → ×1.3
heating_energy    = T_heat × t_heat × 0.1 × complexity_factor
blowing_energy    = P_air × t_blow × 100
cooling_energy    = (60 - R_cool) × 10
f₃ = heating_energy + blowing_energy + cooling_energy
```

**物理意义**：
- 加热段：温度×时间（正相关），复杂度因子放大加热能耗
- 吹制段：压力×时间
- 冷却段：速率越快能耗越低（反向关系）

### 6.4 耐热安全性 f₄

**数学公式**：

```
stress_penalty = C × 20                                        # C=1 → -20°C
heat_temp      = 120 + R_cool × 0.8 + D_wall × 5 - stress_penalty
f₄ = 1 - heat_temp / 200
f₄ = max(0.0, f₄)                                              # 下限 0
```

**物理意义**：
- 耐热贡献：冷却越快 + 壁厚越厚 → 耐热温度越高
- C 引入应力集中折减：复杂形状等效耐热降低
- 最终 MAP 到 [0, 1]（越小表示耐热越好）

### 7.5 C 的影响汇总

| 目标函数 | C 的影响机制 | C=0 → C=1 |
|----------|-------------|-----------|
| f₁ 壁厚均匀度 | 基础标准差升高 + 均匀化增益降低 | 0.12 → 0.18 |
| f₂ 废品率 | 基线升高 + 温度/时间敏感性增大 | 1% → 4%（基线）|
| f₃ 能耗 | 加热能耗放大系数增大 | ×1.0 → ×1.3 |
| f₄ 耐热安全 | 应力集中折减 | −0°C → −20°C |

## 8. 动态约束边界

### 8.1 设计思路原理

根据轮廓复杂度 C 线性插值调整 6 维工艺参数的搜索范围。

### 8.2 两端配置

| 参数 | C=0（简单杯型） | C=1（复杂杯型） | 变化趋势 |
|------|----------------|----------------|----------|
| 加热温度 | [550, 700] | [580, 680] | 区间收窄 + 下界抬高 |
| 加热时间 | [30, 180] | [60, 160] | 区间收窄 + 下界大幅抬高 |
| 吹制压力 | [0.1, 0.5] | [0.2, 0.45] | 区间收窄 + 下界抬高 |
| 吹制时间 | [5, 30] | [10, 25] | 区间收窄 + 下界抬高 |
| 冷却速率 | [10, 50] | [5, 40] | 区间收窄 + 上界降低 |
| 目标壁厚 | [2, 5] | [3, 5] | 下界抬高 |

### 8.3 插值公式

```python
alpha = clamp(C, 0, 1)
for each param k:
    lb = lb_simple[k] + (lb_complex[k] - lb_simple[k]) × alpha
    ub = ub_simple[k] + (ub_complex[k] - ub_simple[k]) × alpha
    bounds[k] = (round(lb, 1), round(ub, 1))
```

C 越高，搜索结果越窄、越偏向高温度/长持压/慢冷却/厚壁厚的保守方案。
