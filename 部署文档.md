# 玻璃杯AI生成可视化设计系统 — 部署与运行文档

本系统基于文生图大模型（默认接入免费、免注册的 **Pollinations.AI**）实现"用户文本驱动的玻璃杯个性化个性化设计"，
并使用 **MOPSO 多目标微粒群算法** 对玻璃热成型工艺参数进行寻优，
同时优化 **壁厚均匀度、成品废品率、加工能耗、耐热安全性** 四项相互冲突的目标。

> **当前默认 provider 已切换为本地 SD WebUI**（见 [feat/local-sd-webui](https://github.com/liu-tr/AI-glass-image/tree/feat/local-sd-webui) 分支），
> 如需恢复 Pollinations 在线模式，请见第七节"更换文生图服务"。

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / 11（脚本按 Windows 编写；Linux/macOS 亦可手动运行） |
| Python | 3.9 及以上（开发环境为 Python 3.14） |
| 网络 | **SD WebUI 模式**：完全离线，无需联网；**Pollinations 模式**：需联网访问 Pollinations |
| 显卡 | **SD WebUI 模式需 NVIDIA GPU（≥4GB 显存）**；Pollinations 模式无需 GPU（云端推理） |

## 四、运行方式

### 方式 A：双击启动（推荐）

直接双击项目根目录下的 **`run.bat`**，脚本会自动：

1. 切换到脚本所在目录；
2. 检测 Python 是否安装；
3. 检测并按需安装依赖；
4. 启动后端服务；
5. 在服务就绪后自动用默认浏览器打开 `http://localhost:5000`。

命令行窗口会实时显示日志，**关闭窗口或按 `Ctrl+C` 即可停止服务**。

### 方式 B：命令行启动

```powershell
python app.py
```

启动成功后终端会显示：

```
* Running on http://127.0.0.1:5000
```

然后手动在浏览器访问 [http://localhost:5000](http://localhost:5000)。

> 注意：数据文件路径 `data/designs.json` 是相对"当前工作目录"的，
> 因此请务必在项目根目录下运行 `python app.py`（`run.bat` 已自动处理）。

## 五、使用流程

1. 左侧输入设计需求（如"简约风格，圆柱形，透明玻璃，适合咖啡饮用"）→ 点击 **生成设计方案**；
2. 系统返回 4 张候选图，点选其中一张；

4. 页面下方展示优化结果：
   - 外观设计 vs 优化后 的四项目标对比；
   - 最优工艺参数表（加热温度、加热时间、吹制压力、吹制时间、冷却速率、目标壁厚）；
   - 算法收敛曲线；
   - 帕累托前沿散点分布。

## 七、图片生成模式说明

系统默认接入 **Pollinations.AI**（免费、免注册、无需 API key）。启动时会探测其可用性：

- **真实模式**：探测成功时，服务端携玻璃行业专用提示词请求 Pollinations，逐张抓取生成的图片并转为 base64 返回前端，得到与输入文本相关的真实 AI 玻璃杯图片；
- **mock 模式**：探测失败时（如断网），自动降级为占位图（来自 picsum.photos 的随机图片），
  与输入文本无关，仅用于在断网环境下演示完整流程。

启动日志会明确提示当前所处模式：

```
✓ 已连接到 Pollinations 文生图服务        # 真实模式
⚠ 文生图服务不可用，使用模拟模式: ...      # mock 模式
```

> **速度说明**：Pollinations 为免费服务，单次生成 4 张图约需 1–2 分钟（高峰排队时更久），
> 服务端已对每张图做失败重试并在彻底失败时兜底占位图，保证前端不出现破图。生成的图片会以 base64 持久化到 `data/designs.json`，再次查看无需重新生成。

### 输入合规预检（input_guard）

系统会在调文生图前**先扫一遍用户 prompt**（`src/services/input_guard.py`），命中工业负面词
（变形/破损/裂纹/碎裂/破碎/脏/丑/损伤/损坏/坏/破 等 17 个中英文词）时，
**不调文生图、跳过 Pollinations 限流**，直接返回一张内联 SVG 灰色提示图（无红斜杠、灰色 ⚠ + "无法生成"），
不污染数据库。正常风格描述（ins风、少女风、复古做旧等）不会被误判。

### SD WebUI 模式（本地推理，推荐）

适用于已下载 B 站**秋葉 A绘世整合包**（或其他基于 AUTOMATIC1111 WebUI 的发行版）的用户。
完全离线、生成速度快、不受第三方服务限流。

#### 1. 准备工作

| 项目 | 要求 |
|------|------|
| 整合包路径 | `F:\sd-webui-aki-v4.10\`（按实际修改 `start_all.bat` 顶部的 `SD_LAUNCHER`） |
| 模型文件 | `stable-diffusion-v1-5-pruned-emaonly.safetensors` 放入整合包的 `models/Stable-diffusion\` |
| 启动参数 | `webui-user.bat` 的 `set COMMANDLINE_ARGS=` 行末尾**必须包含** `--api`（整合包默认已带，可在启动器内确认） |
| 端口 | 默认 7860（被占用时需手动改） |

#### 2. 启动流程

双击项目根目录的 **`start_all.bat`**，脚本会按顺序执行：

1. 拉起 A绘世启动器 (`A绘世启动器.exe`)；
2. **人工操作**：在启动器窗口点击「一键启动」；
3. 脚本每秒轮询 7860 端口，每 5 秒打一次心跳，最长等 5 分钟；
4. 端口就绪后自动 `curl` 探活 `/sdapi/v1/options`，确认 `--api` 已开启；
5. 调起项目自带的 `run.bat` 启动 Flask，并自动打开 `http://localhost:5000`。

控制台会显示：

```
[步骤 1/4] 启动 A绘世启动器: F:\sd-webui-aki-v4.10\A绘世启动器.exe
            请在弹出的窗口里点击"一键启动"按钮
[步骤 2/4] 等待 SD WebUI 就绪 (端口 7860，最长 300s)...
            等待中... 5s
            等待中... 10s
[信息] SD WebUI 已就绪 (耗时 32s)
[步骤 3/4] 校验 API 端点 /sdapi/v1/options ...
[信息] API 端点正常 (HTTP 200)
[步骤 4/4] 启动 Flask 服务...
 * Running on http://127.0.0.1:5000
```

Flask 启动后看到的连接信息是 `✓ 已连接 SD WebUI: http://127.0.0.1:7860/sdapi/v1/txt2img`，
即进入真实推理模式；显示 `⚠ SD WebUI 不可用，使用模拟模式` 则进入 mock（picsum 占位图）。

#### 3. 关闭顺序

为避免显存残留，**按以下顺序关闭窗口**：

1. Flask 命令行窗口 → `Ctrl+C` 或直接关；
2. A绘世启动器窗口；
3. SD WebUI 的 cmd 窗口（看到 `Shutting down` 后再关）。

#### 4. 常见报错速查

| 控制台信息 | 原因 | 解决 |
|------------|------|------|
| `找不到 SD WebUI 启动器: F:\...` | 整合包路径变了 | 编辑 `start_all.bat` 顶部 `SD_LAUNCHER` |
| `API 端点返回 HTTP 404` | 启动参数没加 `--api` | 编辑 `webui-user.bat`，在 `set COMMANDLINE_ARGS=` 行末尾加 `--api` 后重启 |
| `等待中... 300s` 超时 | GPU 驱动问题 / 整合包启动失败 | 手动启动 WebUI 看完整报错 |
| Flask 进入 mock 模式 | 7860 没起来 / API 未开 | 按上两条排查后重启 `start_all.bat` |

### 更换文生图服务（可选）

如需更高质量或更快速度，可接入带 API key 的服务（如阿里云百炼·通义万相、硅基流动等）：
在 `src/services/image_generator.py` 中按现有 `provider` 分支新增对应实现，并在 `app.py` 修改
`GlassImageGenerator(provider=...)` 的初始化参数即可。若要接本地 Stable Diffusion，将 provider
设为 `"sd_webui"` 并传入 `api_url` 即可复用内置的 `txt2img` 对接逻辑。

## 八、数据存储说明

- 所有设计方案与优化结果保存在 `data/designs.json`（UTF-8 编码的 JSON 数组）；
- 文件在首次运行时若不存在会自动创建为空数组 `[]`；
- 读取采用 `utf-8-sig` 编码，可兼容带/不带 BOM 的文件（防止编辑器误加 BOM 导致崩溃）；
- 恢复初始状态：将该文件内容清空为 `[]` 即可。

## 九、常见问题（FAQ）

**Q1：双击 run.bat 闪退或出现乱码 / `'xxx' 不是内部命令`？**
脚本已使用纯 ASCII 编写以规避 cmd.exe 的编码解析问题。若仍闪退，请改用命令行方式 B 运行以查看具体报错。

**Q2：启动报 `ModuleNotFoundError`？**
依赖未安装，执行 `pip install -r requirements.txt`。

**Q3：端口 5000 被占用（`Address already in use`）？**
先按"第六节"停止已有进程，或修改 `app.py` 末尾 `app.run(..., port=5000)` 中的端口号。

**Q4：调用生成接口报 `Unexpected UTF-8 BOM`？**
本版本已通过 `utf-8-sig` 读取修复该问题；如仍出现，请确认 `data/designs.json` 未损坏，可将其内容重置为 `[]`。

**Q5：生成的图片和我输入的文字无关？**
说明当前处于 mock 模式（Pollinations 不可达，通常是断网）。请确认网络连通后重新运行，即会自动切换为真实文生图。

**Q6：输入「变形破损裂纹扭曲」等负面词后，4 张图都变成灰色 ⚠ 提示图？**
这是 **input_guard 工业负面词预检**生效的正常行为。系统会在调用文生图前先扫描 prompt，
命中 9 类工业敏感词（破 / 损 / 碎 / 脏 / 丑 / 坏 / 损伤 / 损坏 等，详见 `src/services/input_guard.py`）
时直接返回 SVG 拒绝图，不污染数据库。请改用「风格 / 颜色 / 杯型 / 场景」描述。

**Q7：4 张图生成 8~45s 才能拿到，是不是太慢？**
Pollinations 免费档对单 IP 的并发上限约 1（HTTP 429 Queue full for IP），
任何"分批并发"策略反而会拖到 130s+。当前实现采用**串行抓取 + 4 张 prompt view 变体**，
单图 2~10s、4 张总耗时 8~45s，是免费档的理论下限。如对速度敏感，可接入付费 key
（通义万相 / 硅基流动）或本地 SD WebUI。

**Q8：Flask 控制台报 `SD WebUI 启用了页面但未开启 API`？**
这是 `test_connection()` 在 `/sdapi/v1/options` 收到 404 时给出的提示，说明 WebUI 跑起来了
但没启用 HTTP API。处理方式：编辑整合包里的 `webui-user.bat`，找到
`set COMMANDLINE_ARGS=` 这一行（可能为空），在末尾加上 `--api` 后保存并重启 WebUI。
秋葉整合包默认就带了 `--api`，一般不需要改。

**Q9：接入 SD WebUI 后 4 张图要 30~60s，本地推理为什么没比 Pollinations 快多少？**
三种情况，按概率排序：
1. **首次启动**：PyTorch 正在编译 CUDA 算子（首次约 30~90s），第二次会快很多；
2. **`--xformers` 未开启**：在 `webui-user.bat` 的 `COMMANDLINE_ARGS` 末尾加 `--xformers` 可提速 20~40%；
3. **batch_size=4 一次出 4 张**：SD 一次出 4 张比 4 次出 1 张要快，但首张要等整 batch 完成。
可加 `--medvram`（4~6GB 显存）或 `--lowvram`（<4GB 显存）参数。

**Q10：整合包路径和脚本里写的不一样？**
打开项目根目录的 `start_all.bat`，修改顶部 `SD_LAUNCHER` 变量为你整合包里
`A绘世启动器.exe` 的完整路径即可，其它逻辑无需调整。

## 十、接口一览（供二次开发参考）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 前端主页 |

| DELETE | `/api/designs/<id>` | 删除设计方案 |
